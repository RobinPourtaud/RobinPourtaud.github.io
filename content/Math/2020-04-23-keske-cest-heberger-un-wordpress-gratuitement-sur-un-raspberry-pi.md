WordPress est un système de gestion de contenu (SGC ou content management system (CMS) en anglais) gratuit, libre et open-source.

WordPress est le CMS le plus utilisé au monde. De part sa simplicité d&rsquo;utilisation et sa liberté de design, sa popularité s’explique d&rsquo;elle même.

Malheureusement, utiliser WordPress pour laisser libre cours à son imagination revient souvent à passer par son porte monnaie. 

Pour un site personnel, 4€ par mois reste raisonnable, mais pour un site d&rsquo;e-Commerce, 45€ semble moins attrayant&#8230;. <figure class="wp-block-image size-large">

<img src="https://keskec.fr/wp-content/uploads/2020/04/image-1024x423.png" alt="" class="wp-image-201" /> <figcaption>Tarifs avril 2020</figcaption></figure> 

Il serait donc intéressent de pouvoir profiter de WordPress gratuitement, et ce sans être limité par les blocages du modèle « gratuit ».

Héberger soit même, son site WordPress semble donc être la bonne chose à faire. 

## Le nécessaire:

  * Un ordinateur sous Linux (dans mon cas, j&rsquo;utiliserais un raspberry Pi 3b sous raspbian). Il faut prendre en compte que cet ordinateur devra rester allumé en permanence&#8230;
  * Une connexion rapide est préférée (pas nécessaire). 
  * Un nom de domaine (potentiellement payant).
  * Une bonne heure :).

## Installer un Serveur HTTP

Avant tout, vérifiez que votre ordinateur est bien à jour:

<pre class="wp-block-code" aria-describedby="shcb-language-29" data-shcb-language-name="Bash" data-shcb-language-slug="bash"><div>
  <code class="hljs language-bash">sudo apt update
  sudo apt upgrade</code>
</div>

<small class="shcb-language" id="shcb-language-29"><span class="shcb-language__label">Code language:</span> <span class="shcb-language__name">Bash</span> <span class="shcb-language__paren">(</span><span class="shcb-language__slug">bash</span><span class="shcb-language__paren">)</span></small></pre>

Pour pouvoir envoyer des pages web aux utilisateurs, nous avons besoin pour cela d&rsquo;un serveur HTTP. Il existe Nginx, mais pour ma part, je vais utiliser Apache2.

<pre class="wp-block-code"><div>
  <code class="hljs">sudo apt install apache2</code>
</div></pre>

<pre class="wp-block-code"><div>
  <code class="hljs">sudo service apache2 start</code>
</div></pre>

Maintenant, vous devriez pouvoir accéder à l&rsquo;adresse http://localhost sur votre ordinateur et voir ce magnifique message:<figure class="wp-block-image size-large">

<img src="https://keskec.fr/wp-content/uploads/2020/04/image-1-1024x535.png" alt="" class="wp-image-207" /> <figcaption>Apache2 Debian Default Page</figcaption></figure> 

<blockquote class="wp-block-quote">
  <p>
    Bien sûr, vous pouvez paramétrer votre serveur à distance via SSH. Dans ce cas, veuillez taper son IP local à la place de localhost&#8230;
  </p>
  
  <cite>Petite info</cite>
</blockquote>

## Ouvrir le port HTTP et HTTPS du Router Internet

Pour rendre accessible votre serveur WEB sur internet, vous devez autoriser les connexions entrantes et sortantes sur le port HTTPS, par défaut le port 443.

Ce paramètrage est accessible sous l&rsquo;onglet « Gestion des ports » sur les Freeboxs, ou sur un onglet « Virtual server » sur un Router Tp-link. Il existe un nombre important d&rsquo;interfaces différentes. C&rsquo;est pourquoi je ne peux vous montrer le moyen d’ouvrir les ports sur tous les router internet.

L&rsquo;idée générale consiste à trouver ce menu, rajouter une redirection, et de la remplir ainsi :<figure class="wp-block-image size-large">

<img src="https://keskec.fr/wp-content/uploads/2020/04/image-5.png" alt="" class="wp-image-938" /> <figcaption>Redirection de port: Freebox</figcaption></figure> 

Faites de même avec le HTTP (port 80) si vous le souhaitez.

## Configurer un nom de Domaine

Votre site web est maintenant accessible sur internet, mais uniquement depuis une adresse IP. Si vous voulez donner accès à votre site aux utilisateurs grâce à une adresse du type www.votre-site.com, vous devez posséder un nom de domaine. Pour cela, vous pouvez acheter un nom de domaine. Par exemple sur le site Google Domain. (il existe aussi des sous-domaines gratuit, comme ceux que propose Azote: <https://www.azote.org/>).

<https://domains.google.com/>

Une fois votre domaine acheté, vous devrez configurer vos règles de type A.

Dans l&rsquo;onglet DNS, vous pouvez rajouter cette règle, c&rsquo;est ici que vous configurer les sous-domaines éventuels de votre site.<figure class="wp-block-image size-large">

<img src="https://keskec.fr/wp-content/uploads/2020/04/image-7-1024x307.png" alt="" class="wp-image-940" /> </figure> 

## Installer un Certificat SSL gratuit pour HTTPS

Optionnel: Si vous souhaitez avoir plusieurs sous-domaines, il est préférable de configurer proprement Apache2.

La façon d&rsquo;avoir un certificat gratuit le plus facilement est d&rsquo;utiliser l&rsquo;outil Certbot, il vous délivrera et configurera seul le certificat SSL. Pour cela, ajoutez le PPA de certbot :

<pre class="wp-block-code" aria-describedby="shcb-language-35" data-shcb-language-name="Bash" data-shcb-language-slug="bash"><div>
  <code class="hljs language-bash">sudo apt-get update
  sudo apt-get install software-properties-common
  sudo add-apt-repository universe
  sudo add-apt-repository ppa:certbot/certbot
  sudo apt-get update</code>
</div>

<small class="shcb-language" id="shcb-language-35"><span class="shcb-language__label">Code language:</span> <span class="shcb-language__name">Bash</span> <span class="shcb-language__paren">(</span><span class="shcb-language__slug">bash</span><span class="shcb-language__paren">)</span></small></pre>

Et installez-le : 

<pre class="wp-block-code" aria-describedby="shcb-language-36" data-shcb-language-name="Bash" data-shcb-language-slug="bash"><div>
  <code class="hljs language-bash">sudo apt-get install certbot python-certbot-apache</code>
</div>

<small class="shcb-language" id="shcb-language-36"><span class="shcb-language__label">Code language:</span> <span class="shcb-language__name">Bash</span> <span class="shcb-language__paren">(</span><span class="shcb-language__slug">bash</span><span class="shcb-language__paren">)</span></small></pre>

Une fois installé, vous pouvez maintenant l’exécuter :

<pre class="wp-block-code" aria-describedby="shcb-language-37" data-shcb-language-name="Bash" data-shcb-language-slug="bash"><div>
  <code class="hljs language-bash">sudo certbot</code>
</div>

<small class="shcb-language" id="shcb-language-37"><span class="shcb-language__label">Code language:</span> <span class="shcb-language__name">Bash</span> <span class="shcb-language__paren">(</span><span class="shcb-language__slug">bash</span><span class="shcb-language__paren">)</span></small></pre>

Vous devriez maintenant recevoir la question suivante :<figure class="wp-block-image size-large">

<img src="https://keskec.fr/wp-content/uploads/2020/04/image-8-1024x408.png" alt="" class="wp-image-941" /> </figure> 

Ensuite, acceptez la redirection. Apache doit maintenant redémarrer et vous pourrez (normalement) voir la page Apache s’afficher à l’adresse : https://votre-site.com

## Télécharger et Installer WordPress

Veuillez prendre en compte qu&rsquo;il aurait été préférable de pré-paramétrer son WordPress en local, mais pour simplifier ce tutoriel, je vais vous montrer comment directement avoir un WordPress accessible depuis le nom de domaine.

Positionnez-vous dans le dossier /var/www/

<pre class="wp-block-code" aria-describedby="shcb-language-38" data-shcb-language-name="JavaScript" data-shcb-language-slug="javascript"><div>
  <code class="hljs language-javascript">cd /&lt;span class="hljs-keyword">var&lt;/span>/www</code>
</div>

<small class="shcb-language" id="shcb-language-38"><span class="shcb-language__label">Code language:</span> <span class="shcb-language__name">JavaScript</span> <span class="shcb-language__paren">(</span><span class="shcb-language__slug">javascript</span><span class="shcb-language__paren">)</span></small></pre>

Téléchargez l&rsquo;archive avec la commande : 

<pre class="wp-block-code" aria-describedby="shcb-language-39" data-shcb-language-name="Bash" data-shcb-language-slug="bash"><div>
  <code class="hljs language-bash">wget https://wordpress.org/latest.zip</code>
</div>

<small class="shcb-language" id="shcb-language-39"><span class="shcb-language__label">Code language:</span> <span class="shcb-language__name">Bash</span> <span class="shcb-language__paren">(</span><span class="shcb-language__slug">bash</span><span class="shcb-language__paren">)</span></small></pre>

Ensuite, extrayez l&rsquo;archive : 

<pre class="wp-block-code" aria-describedby="shcb-language-40" data-shcb-language-name="CSS" data-shcb-language-slug="css"><div>
  <code class="hljs language-css">&lt;span class="hljs-selector-tag">unzip&lt;/span> &lt;span class="hljs-selector-tag">latest&lt;/span>&lt;span class="hljs-selector-class">.zip&lt;/span></code>
</div>

<small class="shcb-language" id="shcb-language-40"><span class="shcb-language__label">Code language:</span> <span class="shcb-language__name">CSS</span> <span class="shcb-language__paren">(</span><span class="shcb-language__slug">css</span><span class="shcb-language__paren">)</span></small></pre>

Un nouveau dossier nommé « WordPress » est apparu dans le dossier courant. Vous devez maintenant indiquer à Apache2 de regarder dans ce dossier :

<pre class="wp-block-code" aria-describedby="shcb-language-41" data-shcb-language-name="JavaScript" data-shcb-language-slug="javascript"><div>
  <code class="hljs language-javascript">sudo nano /etc/apache2/sites-available/&lt;span class="hljs-number">000&lt;/span>-&lt;span class="hljs-keyword">default&lt;/span>-le-ssl.conf</code>
</div>

<small class="shcb-language" id="shcb-language-41"><span class="shcb-language__label">Code language:</span> <span class="shcb-language__name">JavaScript</span> <span class="shcb-language__paren">(</span><span class="shcb-language__slug">javascript</span><span class="shcb-language__paren">)</span></small></pre>

Ensuite, indiquez la localisation du dossier comme ceci : 

<pre class="wp-block-code" aria-describedby="shcb-language-42" data-shcb-language-name="Apache" data-shcb-language-slug="apache"><div>
  <code class="hljs language-apache">&lt;span class="hljs-section">&lt;IfModule mod_ssl.c&gt;&lt;/span>
